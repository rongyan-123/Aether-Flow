三、核心 API 接口
================================================================================

1. GET /api/game?playerId=xxx
   - 读取玩家数据
   - 返回: { player, chatHistory }

2. POST /api/game
   - 发送玩家动作（非流式）
   - Body: { input: string, playerId: string }
   - 返回: { success: true }

3. POST /api/game/action ★核心接口★
   - 流式 SSE 返回
   - Body: {
       input: string,         // 玩家输入
       playerId: string,      // 玩家ID
       mode: "prepare"|"play",// prepare=创建角色, play=正常游戏
       llmConfig: {           // LLM 配置
         apiKey: string,
         baseUrl?: string,
         modelId?: string
       },
       playerName?: string    // 创建角色时传
     }
   - SSE 事件类型:
     event: step        → data: { label: "当前步骤说明" }
     event: text-delta  → data: { content: "流式文本块" }
     event: codex       → data: { name, entry_type, description }
     event: journal     → data: { title, content, entry_type }
     event: reply       → data: { reply, player, deltas }
     event: error       → data: { message }
     event: done        → data: ""

4. DELETE /api/game
   - 删除玩家数据
   - Body: { playerId: string }

四、完整游戏流程测试步骤
================================================================================

【创建角色】
1. POST /api/game/action
   Body: {
     input: "请为我开启修仙之旅",
     playerId: "test-player-001",
     mode: "prepare",
     llmConfig: { apiKey: "xxx", baseUrl: "...", modelId: "..." },
     playerName: "测试修士"
   }
   → 接收 SSE 流，检查：
     - step 事件正常推进
     - reply 事件包含 player 数据
     - player.stats 有默认值（hp:100, mp:50, realm:练气期一层）

     - text-delta 流式输出正常

五、边界情况测试清单（按优先级）
================================================================================

P0 - 必须测（核心流程断裂会导致游戏无法玩）
  □ 创建角色 → 选流派 → 首次剧情生成（完整流程）
  □ 连续 5 轮对话（SSE 流是否稳定）
  □ HP 归零 → 玩家状态变为 DEAD
  □ 背包物品增删（数量是否正确）
  □ 境界突破（Check_Breakthrough SUCCESS/FAIL）

P1 - 重要（功能可用性）
  □ 心境只增不减规则（state_of_mind 不能为负）
  □ 护盾系统（伤害优先扣护盾）
  □ 动态灵力注入（mp_cost 1%~100%）
  □ 图鉴条目写入（codex 正确分类）
  □ 局面生命周期（发酵→高潮→收束→结束）
  □ 伏笔埋下与回收

P2 - 正常（体验优化）
  □ 对话自动总结（每20轮触发）
  □ Search_History 工具调用
  □ 逻辑挑刺层（Critique 自动修正）
  □ 寿元耗尽 → 死亡
  □ 并发请求（乐观锁）

P3 - 边缘情况
  □ 空输入
  □ 非法 playerId
  □ 缺少 API Key
  □ 大模型超时
  □ 向量库为空时的降级
  □ 同时多个玩家操作

六、已知问题 / 待修复
================================================================================

1. 数据库端口冲突：5432 被 my_game_postgres 占用，当前用 5433
2. 向量检索用的是 ILIKE 模糊匹配，不是真正的向量相似度搜索

4. 没有乐观锁（version 字段），并发请求可能导致数据覆盖
5. 死亡后没有轮回/百世书系统（Tombstone 表未实现）
6. 前端缺少寿元枯竭警告动画
